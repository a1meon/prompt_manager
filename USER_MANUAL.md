# 使用手册：一世提示词管理

## 1. 产品定位

一世提示词管理用于沉淀提示词模板，并基于模板变量进行单条与批量生成，帮助将提示词生产流程标准化、可复用、可追溯。

## 2. 核心概念

### 2.1 模板

模板是可复用的提示词正文内容。建议在模板中使用自然语言结构化表达，并在可变内容处引入变量。

### 2.2 变量

变量使用 `{变量名}` 标记，例如：

```
请围绕 {主题} 生成一篇文章，风格为 {风格}，字数约 {字数}。
```

应用会自动识别模板中的变量，并在生成时提示你填写。

## 3. 主要功能与操作

### 3.1 创建新模板

1. 点击「创建新模板」
2. 填写「模板名称」「说明备注」
3. 在「模板正文」中输入内容并使用 `{变量名}` 标记变量
4. 点击「保存模板」

### 3.2 修改模板

1. 在模板列表中选择目标模板
2. 点击「修改模板」
3. 更新内容后点击「保存模板」

### 3.3 导入提示词（Markdown 文件）

在编辑模板弹窗中点击「导入提示词」，选择 `.md/.markdown` 或纯文本文件，内容会导入到「模板正文」。

### 3.4 单条生成

1. 在右侧选择「单条生成」
2. 逐项填写变量值
3. 右侧「生成结果」会实时更新
4. 点击「复制结果」复制当前结果

说明：当存在未填写变量时，对应占位不会出现在生成结果中，生成内容可能不完整。

### 3.5 批量生成

1. 在右侧选择「批量生成」
2. 按行输入变量值（每行代表一组数据）
3. 多个变量可用逗号（中/英）或制表符分隔
4. 生成后可使用顶部「目录」快速跳转到某条结果
5. 点击每条结果顶部的「复制」复制该条结果，或点击右上角「全量复制」复制全部结果

### 3.6 导入/导出模板（JSON）

在模板列表区域可进行导出与导入：

- 导出：将当前模板集合导出为 JSON 文件
- 导入：选择 JSON 文件导入模板（会追加到现有模板集合）

### 3.7 自动更新

应用启动时会自动检查 GitHub Releases 是否存在新版本：

- 如检测到新版本，会在后台下载
- 下载完成后会提示「立即安装 / 稍后」
- 选择「立即安装」会关闭应用并启动安装程序完成升级

## 4. 使用建议（企业场景）

- 建议建立模板命名规范：业务域 + 目的 + 版本（例如：`市场-新品文案-v1`）
- 建议在说明备注中写清楚输入约束、输出格式与边界条件
- 批量生成建议在导入数据前先对变量顺序与数量做校验，减少无效结果

## 5. 常见问题

### 5.1 为什么生成结果里某些内容为空？

对应的变量值未填写或为空，系统会在生成时将该变量占位替换为空字符串。

### 5.2 批量生成提示“变量数量不匹配”

该行数据拆分后的值数量与模板变量数量不一致。请检查分隔符与缺失列。
