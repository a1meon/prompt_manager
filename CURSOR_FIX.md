# 光标样式优化说明

## 🐛 问题描述

用户反馈：点击页面很多位置都会出现鼠标手型光标，但只有在文本输入框的位置才有光标（文本选择光标）。

**问题分析：**
- 原本应该只有按钮等可点击元素显示手型光标 (`cursor: pointer`)
- 但实际上很多非按钮区域 (如模板列表项) 也显示了手型光标
- 文本输入框应该显示文本选择光标 (`cursor: text`)，但表现不符合预期

---

## ✅ 解决方案

### 1. **修改 Sidebar 组件 - 模板列表项**

**修改前：**
```tsx
<div className="group relative p-3 rounded-lg cursor-pointer transition-all">
```

**修改后：**
```tsx
<div className="group relative p-3 rounded-lg transition-all cursor-default select-none">
```

**改进点：**
- ✅ 移除了 `cursor-pointer`，改为 `cursor-default` (默认光标)
- ✅ 添加 `select-none` 防止文本被意外选中
- ✅ 保持悬停背景色变化效果，提供视觉反馈

---

### 2. **添加全局 CSS 规则**

在 `index.css` 中添加：

```css
/* 光标样式优化：只有按钮和真正可点击的元素显示手型光标 */
button,
[type="button"],
[type="submit"],
[type="reset"] {
  cursor: pointer;
}

/* 移除 div 和其他非按钮元素的默认手型光标 */
div.cursor-pointer {
  cursor: default;
}
```

**作用：**
- ✅ 确保所有 `<button>` 元素自动显示手型光标
- ✅ 强制移除 `div.cursor-pointer` 的手型光标
- ✅ 统一全站光标行为

---

## 📊 光标类型说明

### 各种光标的正确使用场景：

| 光标类型 | CSS 值 | 使用场景 |
|---------|--------|----------|
| **默认光标** | `cursor: default` | 普通文本、不可点击区域 |
| **手型光标** | `cursor: pointer` | 按钮、链接、可点击元素 |
| **文本光标** | `cursor: text` | 输入框、可编辑文本区域 |
| **禁止光标** | `cursor: not-allowed` | 禁用状态 |

## 🎯 当前页面的光标分布

### ✅ 正确显示手型光标的元素：
1. **主题切换按钮** - `<button>` 元素
2. **导出按钮** - `<button>` 元素  
3. **导入按钮** - `<label>` 元素 (带 `cursor-pointer`)
4. **"立即创建"按钮** - `<button>` 元素
5. **"修改模板"链接** - `<button>` 元素
6. **删除/编辑/预览按钮** - `<button>` 元素
7. **保存/取消按钮** - `<button>` 元素

### ✅ 显示默认光标的元素：
1. **模板列表项** - 整个卡片区域 (`cursor-default`)
2. **标题文字** - 普通文本
3. **说明文字** - 普通文本
4. **标签页切换容器** - 非直接点击区域

### ✅ 显示文本光标的元素：
1. **变量输入框** - `<input type="text">`
2. **批量输入框** - `<textarea>`
3. **模板名称输入** - `<input type="text">`
4. **说明备注输入** - `<input type="text">`
5. **模板正文输入** - `<textarea>`

## 🔍 技术细节

### Tailwind CSS 类名对应关系：

```tsx
// 默认光标
cursor-default    // cursor: default

// 手型光标  
cursor-pointer    // cursor: pointer

// 文本光标
cursor-text       // cursor: text

// 禁止光标
cursor-not-allowed  // cursor: not-allowed

// 不允许选中
select-none       // user-select: none
```

## 🎨 用户体验优化

### 改进后的效果：

1. **清晰的视觉层次**
   - 手型光标 = 可点击操作
   - 默认光标 = 信息展示区域
   - 文本光标 = 可输入编辑

2. **一致的行为模式**
   - 所有按钮统一显示手型
   - 所有输入框统一显示文本光标
   - 列表项显示默认光标但有悬停效果

3. **减少误操作**
   - 添加 `select-none` 防止文本被意外选中
   - 光标样式准确反映元素功能
